工具

如果你也想开始写作，这些写作工具推荐给大家

写作工具推荐 前面我们提到，如果我们想写一点东西，可以从写日记分享自己这样一件小事情做起，目的就是降低开始写作这个行为的阻力和难度，而后在这个过程中慢慢的去发现一些针对自己的问题，再慢慢去解决。 今天的话题，应该是很多写作者都会遇到的问题——有哪些好用的写作工具！ 虽说 AI 的加持让非常多原本就十分优秀的笔记软件更加的优秀，市面上主打各种独家功能——block 块、双链、无限画布、大纲、AI 辅助……——层出不穷、各有特色。 但我还是秉承着那句话： 事情应该力求简单，但不应过于简单！ ——佚名 做事情力求简单，主要是能够把注意力最大化的放在所做的事情上，减少流程、仪式、过程…对主要事情的影响；但为什么不能过于简单呢？同样的原因，过于简单很多时候会无限趋近于简陋、不充分！ 好比于我们想去一个地方吃饭，步行是最简单、无成本的，但会开车看起来更能让人接受。 所以，接下来我给大家分享的工具，有的很简单，也有的需要一些些学习成本，大家不要看到简单的就说“太简单了，我不要用”；也不要说“太复杂了，我不要学”，这样的态度很不正常。 Markdown 语法 我第一个要给大家介绍的，也是最颠覆大部分人的东西，就是一种写作语法—— markdown ！ 这种语法相较于传统的 word、记事本等等 富文本 软件，更能让我们专注于写的动作，淡化美化这个行为。 常用的 markdown 语法： # + 空格 + 内容：可以表示几级标题，一个 # 级别最高、依次递减 - + 空格 + 内容：可以生成项目符号 数字 + . ：可以生成有序列表 ** 内容 **：可以让内容粗体显示 * 内容 *：可以让内容斜体显示 ---：可以生成一条横线分割内容 …… 这些常见的语法，就可以让我们生成一篇排版挺不错的文章了。 左侧是编辑模式，右边是预览模式——最终呈现的效果。 灵感软件 灵感总是转瞬即逝的，那么需要我们及时的捕捉记录，我推荐少楠的 Flomo 这款软件，全平台同步，并且支持所有主流软件的扩展插件，几乎包含我们任何记录灵感的场合——在外面，随时随地用手机写下来或者语音说出来；在浏览网页时，在扩展插件里也可以随手一记，非常的丝滑。 而且这个软件，也是我认为用来尝试 卡片笔记 的好伴侣，非常的好用，值得推荐。 写作软件 我推荐 Typora，这个软件的设计很简约大方，同时，是我用过的最好的及时渲染软件，写作体验非常的丝滑——所见即所得的感觉真的很棒。 并且后期可以搭配图床软件 upic 或者 picgo 来嵌入永久图片；搭配 pandoc 更好的给自己的文章打标签、做引用资料、做导出。 ...

多设备的文件同步和传输问题

不是每个人都是某个电子品牌的死忠粉，也就意味着「全家桶」这个概念，并没有普及到每个人的生活学习和工作。 我很赞成极客湾的云飞和笔吧的猪王在讨论 Mac 电脑使用体验时说的一句话： 粉产品力，而不要粉品牌。 这是典型的实用主义——谁家的东西做得好、值得买，我就为谁买单。这样的观念也能帮助我们逃离科技公司的「生态圈衍生的护城河」牢笼。你可能需要一台 Windows 打游戏和做科研，但喜欢苹果 IPad 的质感和精美软件，平时旅游喜欢 Vivo 的拍照效果，这样的搭配，是很日常也很有依据的组合。 但脱离生态，注定有一些操作是没那么丝滑简约的。比如我们想要在另一台电脑调用手机的文件、照片等等；比如我们想要把旅游时拍摄的照片和视频导入电脑做美化时。没有了「生态互联」的加持，我们需要学会一些另外的手段，解决一些痛点，提升我们的体验。 文件共享 想要在你所有的设备上，都能访问你最常用的一些文件、照片等等，我推荐大家使用「OneDrive」软件。 论可靠性，它是微软家族工作流里的核心项目，所以大胆放心用；论良心程度，它给予了每个免费用户 5GB 的永久空间，5 GB 对于图片视频这类流媒体文件可能不算啥，但对于常用的学习和办公文件，就这么说吧，用来同步我平时看的文献、书籍还有写过的文稿和常用数据文件，都远远没有达到内存瓶颈，所以可以大胆用。 关于它的使用方法，我只教给大家最简单有用的方法： 首先：「选定」一个 OneDrive 所在的位置，我推荐大家把它「固定」在文件管理器的侧边栏，这样好拿取文件。 接着：我们就可以把我们想要在全平台同步的文件「拖」进刚才创建的那个文件夹，我自己的话，是把近期需要用到的文献、科研数据、写作文稿以及常用的零散文件（比如ppt、课题大纲等等）放在里面。 最后：在手机、电脑和平板上都下载这个软件，登录上同一个微软账号，就可以同步上刚刚上传的文件，可以自如的拿取和存放新的文件。 文件互传 把手机里像视频、安装包等等这样的大文件，用同步的方法就不合适，因为占空间不说，速度也不太满意，而且我们大概率只需要传递这么一次，并不需要同步到所有的设备。 还有的人会说，我用 QQ、WeChat，但事实是它们可能会压缩视频和照片画质、限制同时上传的数量，而我推荐的方法，有更多的自由。 所以，我推荐「Blip」这款跨局域网传输文件的神奇。 相比于大名鼎鼎的「LocalSend」，我更喜欢「Blip」在任何网络环境下，都能够传输任意大小和格式的文件，它并不需要我们的设备连接的是同一个 WiFi，这一点就非常好评。 首先：把这个软件下载到所有的设备上。 然后：给这个软件，登录上同一个邮箱。 最后：就可以任意传输文件了。 我们可以把自己不同设备都更改一个更便于识别的名字，对于我这样的轻微强迫症再合适不过了，简直是不要太整齐。 非常的神奇，非常的值得尝试一下，希望对大家有帮助。
